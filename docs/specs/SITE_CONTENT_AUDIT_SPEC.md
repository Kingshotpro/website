# Site Content Audit — Spec for Next Claude

**Task:** Audit every page on KingshotPro for fabricated or unverified game-specific data. Flag, document, and fix.
**Project:** KingshotPro (kingshotpro.com) — gaming dashboard for Kingshot mobile strategy game
**Token budget:** Up to 800K. Use it. This is credibility-critical work.
**Deliverable:** `docs/AUDIT_REPORT.md` + all fabrications fixed in place.

---

## Why This Matters

A previous AI session generated `guides/hero-guide.html` with completely fabricated hero names: "Ironclad", "Sharpshooter", "Guardian King", "Blaze Fury", "Mystic Sage". **None of these exist in Kingshot.** It was caught by accident while building the hero database.

The rest of the site has never been audited. Any page generated by a previous AI could contain invented hero names, fake stats, made-up event mechanics, wrong numbers. A player who finds fabricated content destroys our credibility forever. There is no recovery from "KingshotPro lies."

**Your job: find every fabrication before a player does.**

---

## Rules of Engagement

1. **Never assume data is correct. Verify or flag.** If you can't verify, mark it UNVERIFIED and let the Architect decide.
2. **"Looks plausible" is not verification.** Cross-reference against known good sources (kingshotdata.com, kingshotmastery.com, kingshotguides.com, whiteoutsurvival.app for WOS-Kingshot overlap).
3. **Hero names, skills, stats, event names, building costs, speedup values — all must be verified.** General strategy advice ("upgrade your furnace early") is fine without verification.
4. **When in doubt, show "?" — never a guess.** Empty is better than fabricated.
5. **Document every finding.** If you fix it, say what it was before. If you leave it, say why.

- **Silent fixes** — document everything in the report, even small fixes
